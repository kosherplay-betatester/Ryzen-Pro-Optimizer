Read this before you tune

Ryzen Pro Optimizer writes to your CPU's SMU registers and runs intense stress tests. These actions can crash your system. By continuing, you accept the following risks:

  • BSOD / hard hangs. Aggressive Curve Optimizer offsets can lock Windows mid-task. If this happens during a write-heavy operation, you can lose unsaved work and corrupt files.
  • Remote-session blindness. If you're tuning over RDP and the system BSODs, you may not regain access until someone physically resets the machine.
  • Worst-case CMOS reset. A profile that boots and immediately crashes can leave you having to clear CMOS (jumper or coin-cell pull) to recover BIOS access.
  • Hardware wear. Running Prime95 at full load generates a lot of heat and pulls high current. Verify your cooling and PSU headroom before long runs.
  • WHEA = stop. A WHEA event during stress testing is a hardware-corrected error. Back off the CO offset immediately — don't keep pushing.
  • Settings are temporary. CO writes live in SMU registers until next reboot. To make a profile permanent, write it to BIOS yourself.

Your safety nets: the Esc panic-reset key, the always-visible RESET CO button, the Safety Guard (auto-aborts on temp/voltage/WHEA breaches), the panic-revert breadcrumb (survives BSODs so the next boot can roll back), and BIOS — which always wins on next power cycle.

Use at your own risk. The authors and contributors are not liable for any damage, data loss, or downtime. If you're not comfortable with that, close this tab now.

2. Test & Autotune Stability

Manual test — runs your current CO values through the selected stress test. If any core errors, the report tells you which one and suggests how much to dial back. You stay in the driver's seat.
Auto-Adjust (autotuning) — CoreCycler runs the stress test and automatically bumps each core's CO offset upward (less negative) whenever that core errors, then re-tests it. Walks each core to its individual stable edge. Set how far it can back off (Max value) and how aggressively it steps (Increment).
  • Starting values = your current per-core CO (read from SMU at test start)
  • Max value = the least-negative offset Auto-Adjust is allowed to reach. 0 means "give up at neutral"; 5 would allow positive offset (rarely useful, usually risky)
  • Increment = how many points to add per error. 1 is precise but slow; 3–5 finds the rough edge faster
  • Useful when you want to walk away and let it find each core's edge autonomously — common end-state: each core at its individual stable minimum
  • Watch the live status — it can take hours per core if errors come early in a long iteration

Default 1 cycle. 3+ cycles recommended for confident stability. Auto-Adjust typically benefits from longer runs (10+) so each core gets multiple chances.

Smart Auto-Adjust uses bisection + telemetry + crash history. See the Tune Theater for live progress.
